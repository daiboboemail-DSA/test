# ✅ 验证 Supabase 配置 - 详细操作步骤

## 🎯 这一步的目的

验证你的 Supabase 配置是否正确，确保应用可以正常连接数据库。

---

## 📝 操作步骤（详细版）

### 步骤 1：打开终端（Terminal）

#### Mac 用户：
1. 按 `Command + 空格键` 打开 Spotlight 搜索
2. 输入：`终端` 或 `Terminal`
3. 按回车打开终端应用

#### 或者：
- 在"应用程序" → "实用工具" → 找到"终端"

### 步骤 2：进入项目目录

在终端中，输入以下命令并按回车：

```bash
cd /Users/daishengan/Desktop/test
```

**说明**：
- `cd` 是切换目录的命令
- `/Users/daishengan/Desktop/test` 是你的项目路径
- 按回车后，终端会进入项目目录

**验证**：如果看到路径变成了 `test`，说明成功了**

---

### 步骤 3：启动开发服务器

在终端中，输入以下命令并按回车：

```bash
npm run dev
```

**说明**：
- `npm run dev` 是启动开发服务器的命令
- 这会启动 Vite 开发服务器
- 需要等待几秒钟

**你会看到类似这样的输出**：
```
  VITE v5.4.8  ready in 500 ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
  ➜  press h + enter to show help
```

**重要**：
- 看到 `ready` 和 `http://localhost:5173/` 就说明启动成功了
- **不要关闭这个终端窗口**（关闭后服务器会停止）

---

### 步骤 4：打开浏览器

1. 打开任意浏览器（Chrome、Safari、Firefox 等）
2. 在地址栏输入：`http://localhost:5173`
3. 按回车访问

**说明**：
- `localhost` 表示本地电脑
- `5173` 是开发服务器的端口号
- 你应该能看到你的网站

---

### 步骤 5：打开浏览器控制台

#### 方法一：使用快捷键
- **Mac**：按 `Command + Option + I`
- **Windows**：按 `F12` 或 `Ctrl + Shift + I`

#### 方法二：使用菜单
- **Chrome/Safari**：右键页面 → "检查" 或 "Inspect"
- 或者：菜单 → "开发者工具" → "控制台"

**你会看到**：
- 浏览器窗口会分成两部分
- 下半部分（或右侧）是开发者工具
- 点击 "Console"（控制台）标签

---

### 步骤 6：检查控制台是否有错误

#### ✅ 配置成功的情况：

控制台应该：
- **没有红色错误信息**
- 可能有一些灰色的信息（这是正常的）
- 没有 Supabase 相关的错误

**示例（正常情况）**：
```
✓ No errors
```

或者：
```
[Info] Connected to Supabase
```

#### ❌ 配置失败的情况：

如果看到红色错误，可能是：

**错误1：Supabase URL 或 Key 未配置**
```
Error: Missing Supabase URL or Key
```

**解决方法**：
- 检查 `.env.local` 文件是否存在
- 检查文件内容是否正确
- 确保没有多余的空格

**错误2：Supabase 连接失败**
```
Error: Failed to connect to Supabase
```

**解决方法**：
- 检查 Supabase URL 是否正确
- 检查网络连接
- 检查 Supabase 项目是否正常运行

**错误3：找不到 .env.local 文件**
```
Error: Environment variables not found
```

**解决方法**：
- 确保 `.env.local` 文件在项目根目录
- 确保文件名正确（前面有点）
- 重启开发服务器

---

### 步骤 7：测试功能（可选但推荐）

如果控制台没有错误，可以测试一下功能：

1. **测试上传案例**：
   - 在网站上点击"添加新案例"
   - 填写信息并上传图片
   - 如果上传成功，说明配置完全正确

2. **检查数据**：
   - 回到 Supabase 控制台
   - 查看 Table Editor → cases 表
   - 应该能看到刚上传的数据

## 🆘 常见问题

### Q1: 终端显示 "command not found: npm"
**原因**：Node.js 没有安装或没有配置到 PATH

**解决**：
1. 检查是否安装了 Node.js：
   ```bash
   node -v
   ```
2. 如果没有安装，访问 https://nodejs.org/ 下载安装

### Q2: 终端显示 "Cannot find module"
**原因**：项目依赖没有安装

**解决**：
```bash
npm install
```
然后再次运行 `npm run dev`

### Q3: 端口 5173 已被占用
**原因**：另一个程序正在使用这个端口

**解决**：
1. 关闭其他可能占用端口的程序
2. 或者修改端口（在 `vite.config.js` 中配置）

### Q4: 浏览器显示 "无法访问此网站"
**原因**：开发服务器没有启动成功

**解决**：
1. 检查终端是否有错误信息
2. 确保 `npm run dev` 命令执行成功
3. 检查终端显示的端口号是否正确

### Q5: 控制台有错误，但不知道是什么
**解决**：
1. 截图错误信息
2. 检查错误信息中的关键词
3. 根据错误信息查找解决方案

如果遇到错误，可以：

1. **截图整个控制台**：

2. **复制错误信息**：
   - 在控制台中右键错误信息
   - 选择 "Copy" 或 "复制"
